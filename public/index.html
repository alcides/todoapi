Documentation

Retrieve items

GET /todos

Should return all items in txt, json or xml format, according to the Accept header.

Creating new items.

POST /todo/9999 name=Document API

Will create a new item with id 9999, text “Document API” and not completed.

May return 405 if already exists, 400 if it has any missing parameter and 201 if everything is okay.

Updating an item

PUT /todo/9999

Will mark item 9999 as completed. If it is already completed, then it will make it uncompleted.

May return 404 if not found or 200 if succeeded.

Delete an item

DELETE /todo/9999

Will remove completely the item 9999 from the todo list.

May return 404 if not found or 200 if succeeded.
